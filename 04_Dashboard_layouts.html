Notebooks with "dashboard" / split cell layouts¶

Intro¶

It's remarkably difficult to get a true dashboard layout without creating a full web app - something which significantly increases the cost and complexity of any project. As a workaround, I have made some custom template files that allow for cell splitting within the notebook. This enables graphs and tables to be displayed side by side.

It is not as flexible as a full dashboard capability, but it should be sufficient for many projects.

Enable splitting cells¶

We can use the Split Cell Notebook extension to enable splitting cells 50:50 in the notebook.

  • First, ensure that you have nbextensions enabled. If you don't, follow the instructions in the guidelines to set up your template
  • Enable Split Cells Notebook Extension
  • A new icon will appear, as shown in the picture below. Click it to split any code cell in the notebook and have them aligned side by side.

And now it's very easy to layout cells side by side simply by toggling that button - just like so!

Some table

Note: The extension only shows code cells as being split when viewing in the Notebook. It is also possible to split Markdown cells, but it won't show up in the Notebook view (it will only show when you export to HTML).
If this is something you really want, edit the cell Metadata and set: {"cell_style": "split"}
If you aren't sure how to edit cell metadata, see 01_Ricardo_template_example.html

Exporting¶

Unfortunately, that's not the end of the story. The standard extension only works in the Notebook view. If you try to export it, you will find that the cells just return to the usual report layout. The way to fix this is by applying a custom template file.

There are a few steps, but it's a one-time only setup.

Exporting with nbconvert¶

Make sure you have the Ricardo stylesheet enabled - use the guide in 01_Ricardo_template_example.html if you aren't sure how to do this.

Now you can use the following command to export your document to html:

jupyter nbconvert name.ipynb --template ricardo_dash_with_TOC
Important!: You must have the Ricardo stylesheet enabled in your notebook for the export to work. The stylesheet contains custom functions that enable rendering of the split cell layout to HTML.
EXTRA IMPORTANT!: Please download the ricardo_stylesheet.css again from the repository if you have been using an earlier version. The new version contains additional code that allows for the exporting. The latest version is always in the repository.

Activate the custom extension¶

You need to run the command jupyter contrib nbextensions install to rewrite the Jupyter config files. Then, restart the notebook server to see your changes.

In [ ]:
!jupyter contrib nbextensions install

You should now see a new Ricardo split cell extension in your extensions list.

  • Activate this
  • Keep the original split cells notebook extension activated too

You can manually edit the cell metadata tags (good for small reports), by using the cell toolbar and setting the "cell_style" property:

  • "cell_style": "split" for 50% width
  • "cell_style": "split_small" for 33% width
  • "cell_style": "split_large" for 66% width

Extra: automate splitting cells¶

For large reports, we don't want to have to manually toggle cells. This extra step is to allow for automatic splitting by injecting some Javascript to edit the tags for us.

Define custom functions¶

Firstly, enable Javascript magic. Put these imports right at the top of your notebook

In [3]:
from IPython.display import Javascript, display
from IPython.core.magic import register_cell_magic, register_line_cell_magic
import json

# Define custom magic to allow editing cell metadata
def _set_style(tags):
    if len(tags)>0:
        assert(tags[0] in ['split', '', 'split_large', 'split_small']), "Set tag to 'split', split_large', or 'split_small'"
        display(Javascript(
            """
            require(['setWidth'], function(setWidth) {
                setWidth(element, %s);
            });
            """ % json.dumps(tags[0])
            ))
    else:
        display(Javascript(
            """
            require(['unsetWidth'], function(unsetWidth) {
                unsetWidth(element, %s);
            });
            """ % json.dumps(tags)
        ))

@register_line_cell_magic
def style(line, cell=None):
    _set_style(line.split())

Add Javascript¶

We also need some custom Javascript to implement the new functions. In a cell below that, insert the following code:

In [4]:
%%js

require.undef('setWidth');
define('setWidth', function() {
    return function(element, tags) {
        var cell_element = element.parents('.cell');
        var index = Jupyter.notebook.get_cell_elements().index(cell_element);
        var cell = Jupyter.notebook.get_cell(index);
        cell.metadata.cell_style = tags;
    }
});
require.undef('unsetWidth');
define('unsetWidth', function() {
    return function(element, tags) {
        var cell_element = element.parents('.cell');
        var index = Jupyter.notebook.get_cell_elements().index(cell_element);
        var cell = Jupyter.notebook.get_cell(index);
        cell.metadata.cell_style = 'center';
    }
});

Use the new magics¶

To update the tag for any cell, use the following commands:

To turn on cell splitting for a particular cell:

# This command with work with the standard extension
%style split # 50:50 splitting

# Only with the customised Ricardo extension
%style split_small # 33% splitting
%style split_large # 66% splitting

Unfortunately, once you've split a cell using the cell magic, it won't unsplit if you just delete the magic command. You need to explicitly turn off the cell splitting by using the following command

%style
